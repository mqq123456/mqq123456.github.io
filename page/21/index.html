读王小波的黄金时代观后感

发表于 2017-09-09

我只能说这是一个操蛋的文学艺术家，写的都是一些再荤不过的段子，文学也许就是如此吧，我终于知道为什么那么多的小说被拍成电影了，本来小说就是写的一场电影
我一直都在告诉自己，你应该要保持学习的态度，多读书，读书才能陶冶自己情操，看视频是没有用的，也许有一天会有意义。
真的不知道怎么说这本书，读完第一篇，感觉还是挺好的，但是越往后越差，我感觉这本书的本意应该是黄金时代，后面的阴阳两面等等都是附属的，也能看书作者能力，我觉得是文革影响了作者的学习，如果作者生活在这个安逸的时代，作者肯定是一个黄段子高手，或者说是类似韩寒，郭敬明之类的作家。
不说了，这本书还是有点遗憾的，以前也读了很多文学作品，感觉这个离文学还是有点距离，本身那晦涩的性爱就是一个很大的问题，但是这也是最关键的问题，作者通过性爱，剖析人性，痛批文革，确实很有个性，也很成功，这是作者很厉害一面，特别是在黄金时代中，但是在其他几篇文章中与黄金时代中显得有点尴尬了，可能是我不懂艺术吧，纯属于自己一个理解哈，也是，自己写的东西为什么要解释呢？
好了，不说了，孩子病了，真的很担心，希望孩子能早日康复，能够健康成长，祈祷，真的老天，祈求孩子能健康成长，就像8号当铺一样，我希望能够让我失去东西，来换孩子健康成长。一种近似歇斯底里，奋不顾身，也许这就是为人父母的感受吧，我知道孩子妈妈跟我一样，一样的担心，她应该更不容易，还要照顾孩子。

如何长知识

发表于 2017-09-09

其实已经活到快30岁了，今天偶尔看了十三邀的节目，突然发现，我如何在这个社会上生活的，突然感觉迷失了自己，从大学毕业，走到这个社会，虽然说还是挺顺利，但是，我究竟得到了什么。毕竟这么多年，学生生涯，好也罢，不好也罢，那是一个成长过程，我们可以回头看自己的无知，就像你现在看孩子掰着手指头在算数一样，你不会觉得可笑，因为那是你的认知过程，但是到今天就不一样了，我们离开了学校这个大熔炉，是的有些人称社会是大熔炉，我估计也这样说吧，毕竟在中国，学生生涯毕竟也是一个漫长的过程，很多人的成长和磨砺也是从这里开始的，估算一下从5岁进入幼儿园，一直刀24，25岁大学毕业，差不多20年的时间，真的，人的一生有几个二十年，但是我们不提那些过去的时间了。
从大学毕业到现在整整3年，3年的时间对于学生生涯可能不长，但是，对于一个人的一生来说，确实是一段很长的时间，我也在这个公司待了快3年的时间了。感慨 万千
从十三邀的节目中，许知远一直在给我们讲述他怎么去通过交谈获取那些心中的疑惑，这些人都是一些文人，或者在某些方面有一定突出认知的人，谈话真的很冲，我感觉到了，我也很欣赏许知远能够这样做，，因为在很多人看来许知远就是一个不会谈话的傻帽，问的问题有的时候很冲，很歇斯底里，但是我想这也是这个节目可贵的地方，一般人做不到这样，特别是我们这代人。
我很理解马东老师说的那句话，世界上只有5%的人在做一些贡献，其他95%的人都是在活着，历史是那5%的人在传播，但是今天不一样了，移动互联网的存在让那95%的人都有了认知这个世界的权利，他们可以去理解，可以去思考，历史是在一直进步，但是进步的方式是很痛苦和缓慢的，历史就是在推到再从新上演的过程中进步。我们今天大部分的矛盾和冲突在历史上都已经上演过了，我们仔细想一想，其实都能找到根源。但是也许只有那5%的人能够去回望历史，发现问题的本质，更多的人是在活着。
再说说我们自己，我们家庭，婚姻，人生成长轨迹，其实很多事情历史中的智者早已经经历过了，但是我们还是要经历一遍，为什么？这就是人生，我们从懵懂的少年，一步步走，一步步的成长，一步步的反思，只有这样才能称为智者，我们不可能从生下来就拥有历史所有的智慧。然后再去扩展新的智慧，机器能做到，人做不到，这才是大自然的轮回之道。
那么关心的问题来了，对于我来说，我是什么样的人，我应该怎么过自己人生，马东老师在30岁的时候就已经对自己人生性格做出了基调，我突然发现，我自己呢？人总是在看别人的人生，忽略了自己，我应该好好反思一下自己人生了。
我突然意识到那句，年轻人没有恒心和毅力的问题了，我发现我真的没有，还是我太年轻了，那我应该怎么做呢？
我最近一直在搞大数据的项目，一直不知道怎么搞，文档写了一堆又一堆，虽然说也越来越有条理，但是，就是这样我突然间迷失自己，这一刻我突然清醒了，这也就是年轻的问题，一个文档你都整理不过来，3天写一个文档，不是因为文档太长，是文档太难写，但是年轻人就是性子太急，难写怎么写嘛。我想这是一个很值得思考的问题。
我一直都在想我要做一个项目，自己有什么想法就实现的项目，但是，到现在来看我都没有实施，每次都是头脑一发热，想一想，然后扔到一边，然后过一段时间再想起来。真的很可怕，这样的我不是真的我。
我想如果想成长，那么现在就要做一些改变了，有的想发不是光想想就结束的，要学着去实施，加油吧，少年。

到目前来说，大数据方面的流程应该是越来约清楚了，虽然说技术上的实现我还是一窍不通，但是，毕竟一个整体的流程出来了，虽然说感觉一些实现还是很复杂，特别是一些算法，数据模型感觉还是有点不理解，但是相信自己，一步一步的来，不要着急，总有一天会学到真正的知识，其实就是这样，一天一天的，我们也在进步，这就让我很欣慰，总的来说，没有虚度这样的时光。
下面还是说说我理解的大数据架构的流程吧
源数据，经过数据清洗入库
分布式存储一个框架，经过strom，批式处理和spark流式处理
模型算法处理，生成应用模型
服务读取模型字段，返回给web浏览器展示

数据源-》数据层-》能力层-》应用层
数据源-》数据整合-》数据存储-》数据计算-》数据分析-》平台服务-》应用层

这样，一条流水线就出来了，虽然说主要的工作都是在华科那里，但是作为一个项目的技术方面，我们应该多去考虑这个项目技术分配，难度，那里会是重点，那里我们需要考虑的多一点，哪里需要考虑的细一点。
总体来说，我个人感觉是，项目总体流程是有了，但是我们没有项目经验，我们也不确定华科是否具备这样的能力，就目前来说，我重点要考虑的可能是中间层的定义，就是数据处理的结果是怎么定义的，我们要对于目前有用，对未来也有用的数据，这才是关键，我们的项目关键是什么，路网的数据能够做什么
1.地理空间的网格化
2.出行路线和路网数据
3.热力图
4.重点地区分析
5.城市概述
有时间多想想表结构吧，表出来了，接下来就好说了。

交通大数据分析

发表于 2017-08-24

到目前来看，我们已经确定这个大数据平台要具有表格中那么多模型，我们根据模型写了相关的静态页面，根据静态页面我们分析前端需要获取的json格式。
那么我们数据底层是什么样子的，我觉的需要一些有数据库开发经验的人去全面考虑一下具体数据库底层要具备哪些字段。这个数据库要具备目前所有模型能力，且对未来一些未开发模型具有可扩展能力。
我简单的想法是，我们可能是要有两个数据库的，1个是大数据的存储数据库，1个是大数据处理分析后的结果数据库。
大数据存储数据库：可能存储的是清洗过后的每个城市的信令数据，路网数据，等等，可能是一个分布式平台。
结果数据库：可能存储的是大数据平台分析、处理后的（每天，或每个时刻的）城市网格，网格人口，网格流动
交通情况，等等
从我们前端要获取的json来看，我们可能要考虑的是这个结果数据库的格式。
我们可能需要的数据库表格样式

1.城市拥堵道路表，
2.拥堵具体路段信息表
2.城市情况表：包括拥堵里程变化，拥堵指数变化，早晚高峰，拥堵预测，等等（也可分成多个表）
3.关键路口表
4.城市网格表
5.城市热力表
6.网格出行表
7.网格poi
8.城市区域表
9.城市区域间交通表

然后根据每个表的功能设计字段

大数据处理分析平台，定时将处理后的结果数据插入到结果数据库中。
前端服务器从结果数据库中进行关联查找，获取对应接口所需要的参数，通过json形式返回给前端。

连接mysql

发表于 2017-08-24

前段时间，闲来无事，研究了一下php连接mysql，今天突然想起来，然后想着再试试还好用不，没想到，今天一试，竟然出问题，到底什么原因，我也一直搞不明白，今天终端提示问题如下：Can’t connect to local MySQL server through socket ‘/tmp/mysql.sock’ (2)
最后，百度了半天原来竟然是这样：mysql服务启动和关闭可以在系统偏好设置里面操作，终端命令如下：
$ sudo /usr/local/mysql/support-files/mysql.server start
$ sudo /usr/local/mysql/support-files/mysql.server stop

原来我一直都没有启动mysql服务，怪不得一直连接不上。。。
