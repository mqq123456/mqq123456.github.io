车计划

发表于 2017-06-08

是的，就要考驾驶证，做个规划，因为，我这是一边上班，一边学驾照，家里还有刚出生的孩子，此时，真的觉得很多事情是会很分心，但是怎么办啊，我也不知道怎么办了，总是要又点条理才行，现在工作上不忙，但是在工作上的时间是固定的，我总不能就这样闲着，什么事都不做吧，一天到晚的就等一个邮件，等一个下班吧，着不是我想要的，我不能这样搞了，我要好好工作了。
是的，说到好好工作，也并非如此，毕竟很多事情要忙，我也需要分身，所以说应该是怎么样合理安排自己的时间才是最重要的，每天早上来到公司，我们就要面临一天的工作，那么合理安排好今天要做什么，这才是本次计划的重点，前段时间算是很好的了解了一下其他的语言，java，Android，php，js，微信小程序啦，都看了一下,但是，说实话，只是泛泛的了解，真的要做的话，还是需要一定行业项目积累的，抱歉，我没有。。。
话说前几日，苹果果断的封闭了热更新，真的让人很头疼，那么怎么办啊。其实就目前来说，热更新算是一个比较成熟的社区了，如果苹果这样封杀的话，会对很多公司造成极大的影响，也不利于科学技术的进步，我觉得，苹果应该是禁止不了的，但是，肯定有办法能够得到一个双方都满意的结果的，大公司肯定会权衡利弊的，但是就目前来说，热更新觉得不适合我们去学习，我们还是本分点的话
但是另一个重要的东西就不得不提了，机器学习，这很重要，其实我觉得这个跟数据挖掘有很多类似的东西，都是属于数据整合方向的，如果这个方向上有突飞猛进的进步的话，应该又会有互联网巨头站出来，但是目前来说，很有可能是被现有的互联网公司垄断的，数据肯定都会是他们的，但是我想说的是，现在所有的公司都重视了线下的重要性，所以说，互联网再牛逼也还是需要实体经济支撑的，对于数据来说，其最终的根源也是来自于线下，所以说现在的互联网公司竞争最激烈的也就是线下。
说了半天，本来是说我自己的事情的，结果拖泥带水的说了一堆，那么我这个车计划要怎么实施尼？想来想去，1.保持现有的工作不变，该忙的时候一定要忙起来，2.积极学习现有的知识，如Android，Swfit，php，java，把这几个知识再巩固一下，每天想着学习一样，来回学习，反复巩固，肯定会有收获。3.避免盲目，减少不必要的心思，特被是现在流行的买车，买房，现在看来还不切实际，一定要控制自己，不要去浏览类似的新闻。4.维护自己的博客，社区，朋友圈，这个也是很重要的一面，我觉得，是时候开放自己，既然自己已经写了，那么就要有发布出来的勇气。5.积极掌握驾车技术，防止考试紧张情况，不要一味的考虑拿到驾照的时间，根据驾校的安排，一步一步的来。6.紧盯孩子成长，孩子还小，成长也是一个慢速的过程，要有耐心，同时要找准自己的落脚点，为孩子未来开辟生存空间。7.驾照学习完之后的事情也要尽快做准备，争取做到一分耕耘，一分收获。

学习总结

发表于 2017-06-08

话说很长时间没有更新了吧，最近遇到一连串的事情，所以说也没时间写东西了。
终于在最终，孩子在5月的8号出生了，真是梨花带雨，满纸荒唐言，一把心酸泪啊。
我在家里呆了有半个多月，孩子出生之前，大学同学来我家玩了一段时间，好好的玩了一把，紧接着就是面临孩子出生，在周一的晚上，反正挺不容易的，我们都熬了一个晚上，到第二天5点才等到孩子到来，然后就是照顾孩子妈妈，以及孩子，我们在医院整整度过了一个星期，我每天都拖着疲惫的身体，在医院里走着，真的很行尸走肉，而且一个星期也不洗澡，天气也挺热了，每天都要出去加工饭，还好，最终我们还是挺过来了，终于和孩子回家了，我记得，我姐夫把我们带回家之后，我就洗澡，然后收拾了一下，吃了点饭，然后就马不停蹄的回北京了，当看到孩子熟睡的样子之后，你就觉得整个心都静下来了，我觉得这就是作为父母意义吧，那种体会，终于来了。
回到北京之后，我也没有经过太多时间的休息，有一个多星期吧，端午节就到了，我当然是又请假回家了，这次在家待了5天，我没有想请太多假，我觉得也不能老是在家里呆着，后来回北京呆的几天，我发现有的时候我真的是没有事情可做，真的，很悠闲，公司状态也百出，我也认真的思考了 下我的未来，我想我真的就这样呆着是不行的，但是我辞职之后也一时半会的找不到理想工作，所以我还是想着在这里呆着吧，什么都不想，肯定也没有什么事，这样也有利于以后在公司混，想到这里我心也就静了下来，我也就想，我没事做，那我为什么不去找点事情做呢？于是，我就想学个驾照了，正好我以前的同事在北京学习过驾照，我也就想了，这回好了，有机会学驾照了，真的，当你想到一件事情的时候，你便有立即把这件事情做好的想法，这个想法是多么可怕，我甚至连报名都没有，也许这也是我浮躁的内心最好的表现吧，我想我是时候改变一下了，毕竟已经是孩子的爸爸了，怎么还是这么的浮躁，应该沉浸下来了，让自己的心沉静下来，不管世界怎么样，我们都要静下来，踏踏实实的，一点一点的学习，一点一点地收获，这才是我们想要的。
有时候总是在想，自己已经过了那个学习的年代，现在学什么都学不下去，但是回头一想，那个活到老，学到老的理念一直在我的心里游荡着，想想都在笑自己，上个周末去驾校报名的时候，遇见一个60多岁，打算学驾照的爷爷，我当时想，老爷爷可以啊，这么大年纪了，还想开车，但是现在想想也对老人家的勇气所折服，这才是真正的学到老来，我怎么了？我这么年轻就觉得自己不行了吗？真的是太愚蠢了。
话说最近一直被驾照的事情折磨着，也是身心疲惫，我说过，我当时想着学驾照，没想到，竟然就这么一直在大脑里盘旋着，然后我的整个大脑就一直这样紧绷着，竟然会这么累，而且每次想到考驾照的时候，我开着车，就相当的紧张，那种手心都会出汗的那种，我真的不知道为什么会这么紧张，这是怎么了，越年纪大，怎么越不敢了吗？着不是真正的我。
也许是经历了太多了，也就怕了，也就不敢冲锋了，也就不敢冒险了，总是佩服那些丛林冒险家，真的是，我正值青壮年，怎么会有害怕呢？想想自己一个人，只身来到北京，我都不怕，刻苦学习，从一个门外汉，到现在这样的工作岗位上，我怕过什么？我何尝不是努力走过来的，我应该可以的，还有什么我做不到的？真的是，我不知道我大学选择学习体育是不是一条正确的道路，我想如果我学习别的行业的话，我是不是会有另一番成就，我想如果没有如果，算了一切都过去了，没有重新来过的道理
话说到孩子，真的是，大学毕业三年了，我三年也啥都没有混出来，算是说能养活自己，但是，说到孩子，真的是挺艰难，不知道在哪里定居，在家里没有人，爸妈年龄大了，也不能照顾孩子了，真的是挺可怕的。好像，一下子，上有老，下有小，真的就这么实现了，我应该怎么办啊。
不管什么时候，我们都应该保持一个平衡的心，人要有毅力，要有恒心，这事最重要的，我们不应该为未来的是而过分分神，也不能不去考虑未来，我们总是说人活着很艰辛，但是回头想想，我们活着，是多么的幸福，所以说，我们还是要脚踏实地的走路，人就是要这样，你不能说自己不行，也不能说自己太牛逼，自己的能力不是自己吹出来的，是自己奋斗出来的，我今天在这样的行业上混着，我总是在想，假如有一天我不这样混了，我在村里当一个普普通通的老师，那又是一个什么场景，我总是这样想着，以至于我都不知道我该怎么奋斗了，是的外面的世界确实是精彩，而且也有的是金钱可以赚，但是我一直在想，我还能这样的赚多久，假如有一天，我赚不下去了，我该怎么办，我上了这么多年的学，我能从学习中积累什么呢？

29个android开发常用的类、方法及接口
1、Throwable类中的getStackTrace()方法，根据这个方法可以得到函数的逐层调用地址，其返回值为StackTraceElement[]；

2、StackTraceElement类，其中四个方法getClassName()，getFileName()，getLineNumber()，getMethodName()在调试程序打印Log时非常有用；

3、UncaughtExceptionHandler接口，再好的代码异常难免，利用此接口可以对未捕获的异常善后；

4、Resources类中的getIdentifier(name, defType, defPackage)方法，根据资源名称获取其ID，做UI时经常用到；

5、View中的isShown()方法，以前都是用view.getVisibility() == View.VISIBLE来判断的；

6、Arrays类中的一系列关于数组操作的工具方法：binarySearch()，asList()，equals()，sort()，toString()，copyOfRange()等；
Collections类中的一系列关于集合操作的工具方法：sort()，reverse()等；

7、android.text.format.Formatter类中formatFileSize(Context, long)方法，用来格式化文件Size（B → KB → MB → GB）；

8、android.media.ThumbnailUtils类，用来获取媒体（图片、视频）缩略图；

9、String类中的format(String, Object…)方法，用来格式化strings.xml中的字符串；

10、View类中的三个方法：callOnClick()，performClick()，performLongClick()，用于触发View的点击事件；

11、TextUtils类中的isEmpty(CharSequence)方法，判断字符串是否为null或””；

12、TextView类中的append(CharSequence)方法，添加文本。一些特殊文本直接用+连接会变成String；

13、View类中的getDrawingCache()等一系列方法，目前只知道可以用来截图；

14、DecimalFormat类，用于字串格式化包括指定位数、百分数、科学计数法等；

15、System类中的arraycopy(src, srcPos, dest, destPos, length)方法，用来copy数组；

16、Fragment类中的onHiddenChanged(boolean)方法，使用FragmentTransaction中的hide()，show()时貌似Fragment的其它生命周期方法都不会被调用，太坑爹！

17、Activity类中的onWindowFocusChanged(boolean)，onNewIntent(intent)等回调方法；

18、View类中的getLocationInWindow(int[])方法和getLocationOnScreen(int[])方法，获取View在窗口/屏幕中的位置；

19、TextView类中的setTransformationMethod(TransformationMethod)方法，可用来实现“显示密码”功能；

Android线性布局

发表于 2017-04-14
  1. 什么是线性布局
    通俗的说感觉起来和线有关，参照线的特点，有么是横向的，要么是竖向的。
    LinearLayout是线性布局控件，它包含的子控件将以横向或竖向的方式排列（通过android:orientation属性来控制），按照相对位置来排列所有的widgets或者其他的containers,超过边界时，某些控件将缺失或消失
  2. 线性布局常用基本属性
  • android:id 这是布局的唯一标识ID
  • android:orientation 他表示的是这个线性布局是采用横向还是纵向布局，通常来说只有两个值：vertical，horizontal
  • android:layout_height 表示当前线性布局的高度
  • android:layout_width 表示当前线性布局的宽度
  • android:gravity 表示所有包含在当前布局中的所有控件采用某种方式对齐（默认左对齐）
  • android:layout_gravity 表示当前线性布局相对于父元素的对齐方式
  • android:background 表示当前线性布局的背景颜色
  • android:layout_margin: 表示外边距，通常表示本控件与父控件四面之间的距离
  • android:padding 表示内边距，通常表示是本元素所有子元素的与父元素边缘的距离，设置在父元素上，比如文字与文本控件的所有距离
  • android:weightSum 权重的总比例
  • android:layout_weight 子元素对未占用空间水平或垂直分布的权重
  • android:baselineAligned 该控件只对能显示text的子控件有效

安卓学习一些类的理解

发表于 2017-04-10

安卓学习还在进行，学习总是不是太深，对于一些东西一知半解的，总感觉不行，这里记录一下吧，希望这样能够加深一下理解

1.FileOutputStream ,FileInputStream
InputStream和OutputStream是抽象类，是所有字节输入流和输出流的父类。这里，我们首 先要分清楚两个概念：
▪ InputStream(输入流):输入流是用来读入数据的。- - - > > >读入
▪ OutputStream(输出流):输出流是用来写出数据的。- - - > > >写出
2.BufferedReader和BufferedWriter
BufferedReader和BufferedWriter大致来说就是一个属于读取文本文件一个属于写入文本文件。
当BufferedReader进行读取文本文件时，会先尽量从文件中读入字符数据并置入缓冲区，
然后如果使用read()方法，会先从缓冲区中进行读取。
如果缓冲区数据不足才会再从文件中读取。
使用BufferedWriter进行写入数据时，并不会先输出到目的地而是先存储到缓冲区中。
如果缓冲区中的数据满了，才会一次对目的地进行写出。

3.String ,StringBuffer ,StringBuilder
String 字符串常量
StringBuffer 字符串变量（线程安全）
StringBuilder 字符串变量（非线程安全）
简要的说， String 类型和 StringBuffer 类型的主要性能区别其实在于 String 是不可变的对象, 因此在每次对 String 类型进行改变的时候其实都等同于生成了一个新的 String 对象，然后将指针指向新的 String 对象，所以经常改变内容的字符串最好不要用 String ，因为每次生成对象都会对系统性能产生影响，特别当内存中无引用对象多了以后， JVM 的 GC 就会开始工作，那速度是一定会相当慢的。
而如果是使用 StringBuffer 类则结果就不一样了，每次结果都会对 StringBuffer 对象本身进行操作，而不是生成新的对象，再改变对象引用。所以在一般情况下我们推荐使用 StringBuffer ，特别是字符串对象经常改变的情况下。而在某些特别情况下， String 对象的字符串拼接其实是被 JVM 解释成了 StringBuffer 对象的拼接，所以这些时候 String 对象的速度并不会比 StringBuffer 对象慢，而特别是以下的字符串对象生成中， String 效率是远要比 StringBuffer 快的
StringBuilder一个可变的字符序列是5.0新增的。此类提供一个与 StringBuffer 兼容的 API，但不保证同步。该类被设计用作 StringBuffer 的一个简易替换，用在字符串缓冲区被单个线程使用的时候（这种情况很普遍）。如果可能，建议优先采用该类，因为在大多数实现中，它比 StringBuffer 要快。两者的方法基本相同。
