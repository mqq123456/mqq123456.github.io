萧红经典小说

发表于 2017-10-13

最近京东上一些便宜东西，我看着挺好，特别是38块钱5本经典大作家的书，我一口气买了10本，算是对这么多年读书爱好的补充吧，即使自己读不完，还能给孩子读，毕竟这些书都是值得读的，都是非常有地位的书籍。不像一些现代文写，小说，那样玄幻，无厘头。
那天看许知远采访西川，我感触很多，文人说的东西就是需要我们去思想一下，他说至少这么多年了，中国一致都是在现代文学，他认为90年代的东西也都是现代文学，我觉得也是，我深刻认识到那些远在抗战时期的作家们，他们写的东西，一如今日，我依然认为那还是现代文学。至少在我心里还是这样认为的，我不认为那些玄幻的东西能走多远。
还有我记得谁曾经说过，历史就是在不断的遗忘然后再现，真正留下来的东西很少很少。至少在那个时候人们也不会认为这个东西会流传下来。就像爱因斯坦，当时就是个科学家，只是当时流行科学，而已，很多人看爱因斯坦，就感觉是一个普通的科学家一样，但是当那些思想流传下来之后，那些千古不变的真理变成现实之后，我们发现，爱因斯坦，那就是神啊。
好了，废话不说，我认为萧红就是一个这样的作家，我看萧红写的小说，我能深刻体会到那种深入骨髓的痛，你看着书，你在咬牙，你觉得哪种痛仿佛就在强加在你的身上，我感觉这是我看到写的最朴素、最真实、最深刻的小说了。书本来就是这样，谁写的真是谁就会打动读者，我感觉萧红的小说有种女性的天赋。小说还是短小精悍的好，我感觉萧红在处理长一点的小说的时候确实有点缺憾。感觉不够完整，内容断片的情况比较大。
哈哈，我怎么能评论作者呢，真是不应该。不说了，只希望多读书，对自己能有好的帮助吧。

地址：http://download.geofabrik.de

相关安装代码

wget http://download.geofabrik.de/europe/germany/berlin-latest.osm.pbf

安装说明，安装wget：sudo brew install wget 安装Homebrew ：ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

3安装osrm-backend Server 端

docker run -t -v $(pwd):/data osrm/osrm-backend osrm-extract -p /opt/car.lua /data/berlin-latest.osm.pbf
docker run -t -v $(pwd):/data osrm/osrm-backend osrm-contract /data/berlin-latest.osrm

运行在5000端口

docker run -t -i -p 5000:5000 -v $(pwd):/data osrm/osrm-backend osrm-routed /data/berlin-latest.osrm

请求示例：

http://127.0.0.1:5000/route/v1/driving/13.388860,52.517037;13.385983,52.496891?steps=true

安装web端用于显示地图，运行在 9966 端口：

docker run -p 9966:9966 osrm/osrm-frontend

一年一度的生日又到了，也很正常，我每年都很平凡的度过，今年也不例外，从小就不过生日，所以也不会有什么抱怨，也许这就是穷人的好处，你没有经历过富裕，所以你也不会有富人的抱怨，哈哈。
昨天QQ上就有很多人给我送生日祝福，我觉得这个功能还是挺有意思的，他不会让你选择是过阳历的生日还是阴历的生日，所以每次你收到的祝福很可能就不是你阴历的祝福，我们都知道，在中国那个历史上，人的生日都是阴历的，根本就没有阳历的，所以，很尴尬了，但是这么多年了，QQ作为一个根深中国大陆的应用，一直都没有尝试去改变这种现象，让我很费解，我想要么就是这帮子产品经理，太先进了，他们过的都是洋人，阳历的生日，所以不会想到13亿百姓的实际情况，但是我想这种情况应该不会持续这样长的时间的，我猜测更大的可能是QQ故意营造这样一个bug，给自己也是给了所有的用户，让人每次过生日的时候都要很特别，让你对QQ有一个深刻的印象，这真的很奇妙，谁说残缺不是美的。
好了，废话不说了，我究竟要做什么啊，我想我在十月一日之前我已经搞完了php的开发，虽然还有很多bug，我也不去想，但是，这两天给我的压力确实挺大，我想我真的不知道怎么做了，我也很累，很多东西我又写，又要去操心，我也不知道该怎么做了，毕竟这些事我都不会做，我真的很头疼，最近让我研究OSRM，我百度了两天，我头都大了，终于整理出了文档，我想我也就到这一步了，哈哈，再怎么搞我也不知道了。
下一步，我去想一想怎么搞出来openstreetmap的路网数据，争取能对下一步的计划有很好的进步，加油吧。
