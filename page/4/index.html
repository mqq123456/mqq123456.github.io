源数据，经过数据清洗入库
分布式存储一个框架，经过strom，批式处理和spark流式处理
模型算法处理，生成应用模型

数据源-》数据层-》能力层-》应用层

那么关心的问题来了，对于我来说，我是什么样的人，我应该怎么过自己人生，马东老师在30岁的时候就已经对自己人生性格做出了基调，我突然发现，我自己呢？人总是在看别人的人生，忽略了自己，我应该好好反思一下自己人生了。
我突然意识到那句，年轻人没有恒心和毅力的问题了，我发现我真的没有，还是我太年轻了，那我应该怎么做呢？

发表于 2017-08-24

到目前来看，我们已经确定这个大数据平台要具有表格中那么多模型，我们根据模型写了相关的静态页面，根据静态页面我们分析前端需要获取的json格式。
那么我们数据底层是什么样子的，我觉的需要一些有数据库开发经验的人去全面考虑一下具体数据库底层要具备哪些字段。这个数据库要具备目前所有模型能力，且对未来一些未开发模型具有可扩展能力。
我简单的想法是，我们可能是要有两个数据库的，1个是大数据的存储数据库，1个是大数据处理分析后的结果数据库。
大数据存储数据库：可能存储的是清洗过后的每个城市的信令数据，路网数据，等等，可能是一个分布式平台。
结果数据库：可能存储的是大数据平台分析、处理后的（每天，或每个时刻的）城市网格，网格人口，网格流动
交通情况，等等
从我们前端要获取的json来看，我们可能要考虑的是这个结果数据库的格式。
我们可能需要的数据库表格样式

1.城市拥堵道路表，
2.拥堵具体路段信息表
2.城市情况表：包括拥堵里程变化，拥堵指数变化，早晚高峰，拥堵预测，等等（也可分成多个表）
3.关键路口表
4.城市网格表
5.城市热力表
6.网格出行表
7.网格poi
8.城市区域表
9.城市区域间交通表

然后根据每个表的功能设计字段

大数据处理分析平台，定时将处理后的结果数据插入到结果数据库中。
前端服务器从结果数据库中进行关联查找，获取对应接口所需要的参数，通过json形式返回给前端。

连接mysql

发表于 2017-08-24

前段时间，闲来无事，研究了一下php连接mysql，今天突然想起来，然后想着再试试还好用不，没想到，今天一试，竟然出问题，到底什么原因，我也一直搞不明白，今天终端提示问题如下：Can’t connect to local MySQL server through socket ‘/tmp/mysql.sock’ (2)
最后，百度了半天原来竟然是这样：mysql服务启动和关闭可以在系统偏好设置里面操作，终端命令如下：
$ sudo /usr/local/mysql/support-files/mysql.server start
$ sudo /usr/local/mysql/support-files/mysql.server stop

发表于 2017-08-10

在我不断的研究ECharts的同时，项目给我们定了一个web静态页面的展示，想让我们写一个类似网站的页面，然后把我们所有的产品模型都涵盖过来，这我怎么会写尼？前端的开发我不会啊，哈哈，终于，我们沈阳的团队起作用了，沈阳过来一个开发人员，帮我们写前端页面，然后我把自己写的ECharts图标放进页面里面，就这样简单，说起来简单，做起来会不会简单尼，其实还是遇到过一点困难，但是总体来说还是比较的流畅，我们竟然花了不到3天的时间就搞定了这样一个页面，确实感觉挺厉害。自我感觉良好啊，哈哈，一时兴起，买了本书，准备陶冶一下情操。
好了，废话少说，还是讲一下开发流程，沈阳同学负责界面的搭建，然后我研究ECharts图标怎么绘制，假数据怎么写，挺快他就开发出了页面，然后就是我要把我的图标放在他写的dom中，确实是出现了一点问题，我们一直在研究，但是放在他的dom上的图标就是显示不出来，这确实很尴尬，因为我在本地都能显示的出来的，最终，他认为是他使用一个插件导致的，最后他更换了一个插件，竟然就好了，真的是很愉快的就都显示出来了，接下来的步骤就是我把这些东西都改成js，让他去引用就可以了。
还有一个很严重的问题就是地图的加载，我一直认为地图的加载是要用localhost访问的，但是我的同事竟然在本地就访问了，这真的很神奇，但是也不考虑这么无厘头的问题了，我初步怀疑是有些浏览器对于报错的限制比较严格导致的。
就这样，紧紧使用2天多的时间，一个静态页面网站就写完了，感觉前端开发确实很快，很便捷。
