小思考

发表于 2018-06-05

今天也是没有什么重要的事情，昨天晚上的苹果开发者大会，发布了很多操作系统的优化，没有一件硬件产品，所以，我想，这个世界可能真的就是在变慢，这样感觉也挺好，起码，我们一时半会还没有被淘汰，但是，当我们没有事情的时候，我们应该干点什么呢，我想，是不是应该写一个app去实现一些东西，这个真的是一个我一直以来都想要去尝试的东西，因为搞这个开发。真的是你闲下来的时候就会手痒痒，这真的是一个让人头疼的问题，但是，其实创新是一件最让人头疼的问题，这就是一个麻烦了，我想我是否能够做一个东西，能够清楚的了解很多东西，就像是一些游戏，我能清楚的知道这个东西应该是什么，应该怎么做，应该怎么样去做，我想做一个工具类的东西，比如说现在高考就要结束了，那么大学是哪个好，专业选哪个好，真的是一个让人头疼的问题，那么我们能不能用一个攻略的东西，让这个事情变的好一点，我想是不是应该有个东西叫做百事通，这个真的是一个让人头疼的问题。
我突然想到一个很有意思的问题，既然现在游戏这么火爆，微信又凭借一个朋友圈火爆四射，那么我们能不能就是来模仿一个微信和王者荣耀，就模仿一把腾讯如何。
现在简单的想法是这样的，我们做一个类似微信发现页面的功能，内容是王者荣耀，这样就能够很好的通用性，然后再加上很多直播等内容的丰富，能不能做到比王者荣耀助手好很多的功能，我现在看王者荣耀助手，感觉很无聊，感觉一点都没有用，还有就是你的很多功能不能得到很好的发展。
现在我定义一下，这么一个app，他首先的定位是工具类。然后能够直播，然后能够慢动作分析。
我想现在像nba一样，一场比赛都会有很多慢动作分析，那么我们能不能分析局势啊，我们能不能把王者荣耀做在沙盘上进行演示？？
还有一个最重要的问题就是，腾讯肯定不会为我们提供这么多的权限，我们也不可能得到王者荣耀用户数据，比如账号名称，等等
但是，我们有英雄，就像我们是一个NBA球迷一样，你肯定有支持的球队，或者你肯定有喜欢的球员，那么这就是一个入手点。
我们是一个荣耀爱好者，那么你肯定就会有你喜欢玩的位置，喜欢玩的英雄。那么这就是入手点。
好了，应用有了，那么你能为英雄做什么呢？这就是我们的功能，说白了，每个爱好者都是想玩好游戏，但是全部的游戏攻略都普遍存在一个问题，就是对游戏认知太浅，感觉就不是一个会玩的玩家写出来的东西，这真的是一个让人头疼的问题。我也不是一个会玩的玩家，我怎么写这些东西呢？？
还有就是我们怎么样才能写好一本书，我曾经多次强调过，一个好的app就应该是一本书，一本很有诱惑的书，你打开之后就爱不释手的感觉，这真的很重要。但是我这个认知比较浅的人，真的能做到吗？
好了，也许大千网友能做到，就好了，认知很重要。我可以向所有人发起这个认知活动，应该是有好心人能够给到一个最好的答案。应该有钱就有人出来贡献吧。
好了，人物已经出来了，但是，就像书一样，我看完了，那么我下次再看的时候又能给我带来什么呢？更可怕的是，有的时候有些人根本不喜欢看第二遍。
这是一个问题，我想我们可以通过很多友好的东西来去做这些东西，我们可以添加一些东西，比如说书签，可以让用户自己编写一些东西，让用户能够更清楚的认识到自己的不足。
就像打篮球一样，我们每次都想找几个人打，每次打完，都没有发现自己有什么进步，这真的让人头疼。

一转眼，孩子已经一岁了，感觉时间很快，说实话我真的没有怎么陪伴孩子，五一的时候我请了三天假，跟着五一一起，相当于在家呆了一个星期，差不多跟过年也差不多了，在家呆了那么久，孩子始终都没有学会叫爸爸，这确实很难，也说明了我陪伴孩子的真的是少。
以前我想着孩子大一点，我带着孩子来北京，我们一家人能在一起，从我上次在家里呆了一个星期之后，我这种想法彻底破灭了，孩子正在成长的时候，孩子喜欢玩了，这个时候每天都希望出去转转，真的是不能在家里窝着的，但是如果在北京的话，真的是只能在家里窝着，因为我要去上班，这真的是一个头疼的问题，一方面想要陪伴，一方面想让孩子自由。
好了，不说那么头疼的问题了，现在孩子真的是越来越淘气了，有时候感觉这孩子特别的娇气，动不动的就会哭，这真的让我很失望，我不希望小小年纪就有多坚强，但是也不能动不动就哭，这真的是太娇气了，更可气的是不吃饭，真的是，我看那么多孩子，都胖嘟嘟的，这孩子真的是一点都不胖，虽说不是多瘦吧，肉也挺结实的，但是，我真的是觉得应该要好好吃饭的，对于饮食上的问题，我觉得真的是要好好的注意，不能小小年纪就惯成不好的饮食习惯，这真的不应该是一个农村孩子应该有的体制。淘气其实我觉得还是可以理解的，毕竟是孩子嘛，但是真的不希望坏的习惯早早的出现在孩子身上，我不能在家里陪伴孩子，说实话，孩子的成长，我至今都没有做任何工作，但是我真的希望孩子能养成好的习惯，这对将来真的很重要，我真的不知道该怎么做了，如果让孩子来北京的话，确实是太痛苦了，但是在家里的话，我真的是没有时间去教育孩子的，这是一个很大的问题。
我结下来讲的问题是独立的问题，我想孩子你已经一岁了，至少也需要慢慢的减少母亲的陪伴了，有的时候要学会自己玩乐，学会跟孩子玩乐了，也要学会母亲不在的时候，不要哭哭啼啼了。我其实很像给孩子买个婴儿床，这个问题我觉得很重要，孩子慢慢的长大了，要学会独立了，不致于早早的独立生活，但至少要有一些独立生活的行为，自己睡觉我觉得应该是一个常识，我问题我同事，他家的孩子很早的时候就自己睡，我觉得这样很好。
说到睡觉，这孩子真的是不省心，睡觉的姿势真的是千奇百怪，我真的很无语，睡着的时候需要翻来覆去的半天，最后轰然趴倒在被窝里，就这样跟趴着睡着了，我们再把她弄到被窝里，但是不一会她就会把被子蹬到一边去，然后睡着睡着，竟然会横过来，头一个劲的往上顶，顶到枕头里，顶到整个头都仰着，歪着，那姿势，我觉得需要一定的柔韧，反正我是做不到。
家里床是很大，让孩子自己睡一个床，有时候觉得确实有点占地方，这真的是一个头疼的问题。我想这个床真的是应该买的。
我没有照顾孩子，但是从目前来看，孩子的身上确实有一些小的毛病，我知道媳妇一直很疼爱孩子，其实照顾孩子真的不容易，在这里说一句，孩子她妈妈辛苦了。
话说回来，有些问题，我们看到了，但是我们不知道怎么解决。也许只有孩子的妈妈能解决吧，我只能是一个旁观者，有的时候话多了，反而是显得的不干活，还净是一些事。

Android开发代码规范

发表于 2018-03-21

最近公司安卓项目比较紧，所以让我去帮忙写一点安卓，我也有幸能写一下安卓，感觉对我有很大的帮助，这种机会是不能少的，我赶紧学习了一番。现在需求差不多了，感觉写的不是太好，但是也不至于太差，现在是时候总结一下了。
1.类中对象以m开头，后面跟对象属性，然后是对象名称。
2.类中方法以set，get为主，所有m对象无法直接访问，都是通过set，get方法间接访问。
3.selector，这个是在Android中发现的一个比较有意思的东西，真的特别的好，非常实用，可以很好的更改UI，让一个控件显示很多不同的风格。
4.安卓中很少写宽高，很少直接写，都是间接写，或者用padding撑起来。
5.安卓中的生命周期还是需要了解的，安卓中所有对象都是围绕着activity来的

Android Studio目录结构

新建工程项目后AS的Product目录结构如下所示：
.idea：//AS生成的工程配置文件，类似Eclipse的project.properties。
app：//AS创建工程中的一个Module。
gradle：//构建工具系统的jar和wrapper等，jar告诉了AS如何与系统安装的gradle构建联系。
External Libraries：//不是一个文件夹，只是依赖lib文件，如SDK等。

新建工程项目后AS的Module目录结构如下所示：

build：//构建目录，相当于Eclipse中默认Java工程的bin目录，鼠标放在上面右键Show in Exploer即可打开文件夹，
编译生成的apk也在这个目录的outs子目录，不过在AS的工程里是默认不显示out目录的，就算有编译结果也
不显示，右键打开通过文件夹直接可以看。
libs：//依赖包，包含jar包和jni等包。
src：//源码，相当于eclipse的工程。
main：//主文件夹
java：//Java代码，包含工程和新建是默认产生的Test工程源码。
res：//资源文件，类似Eclipse。
layout：//App布局及界面元素配置，雷同Eclipse。
menu：//App菜单配置，雷同Eclipse。
values：//雷同Eclipse。
dimens.xml：//定义css的配置文件。
strings.xml：//定义字符串的配置文件。
styles.xml：//定义style的配置文件。
……：//arrays等其他文件。
……：//assets等目录
AndroidManifest.xml：//App基本信息（Android管理文件）
ic_launcher-web.png：//App图标
build.gradle：//Module的Gradle构建脚本

AS的工程根目录下的build.gradle文件：

buildscript { //设置脚本的运行环境
repositories { //支持java依赖库管理（maven/ivy等）,用于项目的依赖
//mavenCentral() //仅仅是不同的网络仓库而已
jcenter() //推荐使用这个仓库
}
//依赖包的定义。支持maven/ivy、远程、本地库、单文件，前面定义了repositories{}jcenter库，使用jcenter的依赖只需要按照
//类似于com.android.tools.build:gradle:1.0.0-rc2，gradle就会自动的往远程库下载相应的依赖。
dependencies {
classpath ‘com.android.tools.build:gradle:1.0.0-rc2’

// NOTE: Do not place your application dependencies here; they belong
// in the individual module build.gradle files
}
}
//多项目的集中配置，多数构建工具，对于子项目的配置，都是基于继承的方式。Gradle除了提供继承方式设置子项目，还提供这种配置
allprojects {
repositories {
jcenter()
}
}

AS的工程根目录下的settings.gradle文件：

include ‘:app’ //module include ‘:my_lib’ //module(build as lib)
AS的工程根目录下的Module的build.gradle文件（此处以一个简单的Lib module的gradle为例）：

//plugin在AS里取值一般为’com.android.library’或者’com.android.application’
apply plugin: ‘com.android.library’ //构建为lib

compileSdkVersion 17 //编译需要SDK版本
buildToolsVersion “19.1.0” //SDK Manager确定本地安装该版本才可以

targetSdkVersion 17 //目标版本
}

buildTypes { //编译项
release {
minifyEnabled false
proguardFiles getDefaultProguardFile(‘proguard-android.txt’), ‘proguard-rules.txt’
}
}
}

dependencies { //依赖支持
compile ‘com.android.support:support-v4:18.+’
}
