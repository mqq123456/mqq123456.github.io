下面是一个示例，它展示了当异步API准备好时，作用域如何允许您覆盖一个虚拟提供者。

Riverpod允许你创建一个ProviderScope，它可以访问父作用域中所有提供者的状态。
return ProviderScope(
parent: ProviderScope.containerOf(context),
child: const AlertDialog(
content: CounterDisplay(),
),
);
